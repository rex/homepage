# Agent Guidance

## Scope
These instructions apply to the entire repository.

## Documentation Policy
- Do **not** delete documentation unless the file's corresponding feature or artifact is also removed.
- Reflect every code change in the relevant documentation files.
- Document all new features or functionality thoroughly to aid future maintenance.

## Code Style Notes
- Prefer descriptive component and utility names that clarify intent.
- Keep Tailwind design tokens centralized in `tailwind.config.cjs`; update documentation when adding or changing tokens.
- Maintain semantic HTML structure and ensure accessibility considerations (labels, alt text) remain intact.
